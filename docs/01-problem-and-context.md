# 01 · Problem & Context

## 🌍 Business Context

某北美 AI 教育出海产品的获客模式：**雇佣 600+ 北美校园大使（Creator）** 在 TikTok / Instagram / YouTube 拍短视频引流。这条链路由 **10 人内部 Coach 团队**管理，每周处理 2000+ 视频。

## 🎯 The Concrete Problem

随着大使规模从 50 → 200 → 300+，Coach 团队面临一个**反复出现的决策痛点**：

> **「谁该重点培养？谁该清退？」**

这个决策原本完全凭经验做：
- "感觉这人最近不行了" → 清退
- "这人看起来有潜力" → 继续投入资源

带来几个问题：

| 问题 | 后果 |
|------|------|
| **缺乏量化标准** | 不同 Coach 判断不一致；管理层无法 review 决策合理性 |
| **300+ 大使无法人工逐个评估** | Coach 只能看自己 Top 关注的几位，其余靠"模糊感觉" |
| **没有早期识别机制** | 等"看起来不行"时已经投入了 6-8 周培养成本 |
| **清退决策被情感影响** | "Ta 上周突然出了个 5K，再给一个月吧" → 长期不下决心 |

## 🔍 What I Set Out To Build

不是"做一个 PPT 报告"，而是一个**能直接驱动 Coach 团队决策的产品级评分系统**：

1. **可批量评估**：300+ 大使一次跑完，分钟级别
2. **量化标准**：6+ 个维度，每个维度可解释
3. **分级 + 推荐动作**：Coach 看完直接知道下一步做什么
4. **支持迭代**：参数和阈值可调，随着业务变化重新校准

## 🧠 Key Design Decisions

| 决策 | 理由 |
|------|------|
| **基于竞品数据反推规律**（而非自家数据）| 自家数据样本量不够，规律不显著 |
| **以"30 视频拐点"为方法论核心** | 详见 [02-data-insight.md](02-data-insight.md) |
| **剔除 Trend / Warm-up 视频** | 这两类视频不代表内容能力（一个是蹭热点、一个是热身），评分需要看真实"模板视频"表现 |
| **引入 Phase 概念**（Early / Mid / Potential）| 让新人和老人用不同评分标准，避免误伤新加入的潜力股 |
| **允许 Coach 复核** | 评分是辅助决策，不是自动清退；最终决策权在 Coach |
